自定义GitHub Pages域名

这时候会发现博客是github.io的二级域名，如果想要博客使用自己的域名，该怎么办呢？

在source目录中新建CNAME文件，文件中写入自定义的域名blog.gzj.me，生成静态文件并部署到GitHub Pages。

DNS中添加一条CNAME记录，把blog.gzj.me指向ghijnuuz.github.io。

等几分钟DNS生效后，就可以通过http://blog.gzj.me访问博客啦。

让GitHub Pages的自定义域名支持HTTPS

配置好了自定义域名，又会发现另外一个问题，HTTP方式可以正常访问博客，但是HTTPS因为证书签名的关系，会不能访问。

这个问题可以通过使用Cloudflare解决。

  • 使用Cloudflare的DNS功能解析域名，并开启Proxy(CND)功能，所有流量通过Cloudflare进行转发。
  • 打开Crypto设置，把SSL设置成Flexible。
  • 打开Page Rules设置，添加一条记录，把http://blog.gzj.me全部用Forwarding URL 301 - Permanent Redirect跳转到https://blog.gzj.me。

这样通过Cloudflare中转后，就可以使用HTTPS访问博客了。

自定义Coding Pages域名

Coding Pages同样提供绑定自定义域名功能，这个功能需要银牌会员才能使用，而银牌会员只需要完善Coding账号的个人信息就可以获得，几乎没有门槛。

DNS中添加一条CNAME记录，把blog.gzj.me指向pages.coding.me。

在Pages 服务设置页自定义域名中绑定域名，等DNS生效后就可以使用自定义域名访问博客。

HTTPS访问

Coding Pages提供HTTPS访问功能，只需在Pages 服务页面SSL/TLS 证书中申请证书，这样会自动通过Let’s Encrypt申请到SSL/TLS证书。

申请到证书后，打开强制 HTTPS 访问开关。
