使用Lombok简化Java代码

发表于 2017-10-20 |

Lombok简介

Lombok项目是一个java库，可自动添加到编辑器和构建工具，加快java的开发。再也不用写getter或equals方法。提早使用java未来的功能，比如val等等。Lombok通过在编译时期将注解替换成具体的代码，实现了这些功能。

安装

Maven

要配置任何构建工具使用Lombok，必须指定编译源代码需要使用Lombok依赖关系，但在运行、测试、部署代码时并不需要Lombok。通常称呼这种为provided依赖。下面介绍如何将Lombok和Maven构建工具集成在一起。

添加Lombok到pom文件，把Lombok作为provided依赖项，添加到dependencies节点中。

IntelliJ IDEA

Jetbrains IntelliJ IDEA兼容Lombok。

在IDEA的Plugins中搜索Lombok Plugin，点击Install进行安装，安装完成后重启IDEA。

稳定功能

val

简单的声明final局部变量。

可以使用val作为局部变量的声明类型，而不用写出实际的对象类型。这么做的时候，会从初始化表达式推断出类型。这个局部变量也会成为final局部变量。这个功能仅适用于局部变量和foreach循环，不适用于字段。初始化表达式是必需的。

val实际上是一种在Lombok包中存在的类的类型。必须import val才能工作（或者使用lombok.val作为类型）。

Lombok

翻译成Java

@NonNull

不用再担心NullPointerException。

可以在方法或构造方法的参数上使用@NonNull来让Lombok生成null检查语句。

null检查语句if (param == null) throw new NullPointerException("param");会被插入到方法的最上方。对于构造方法，null检查语句会被插入到this()或者super()后面。

如果方法顶部已经存在null检查语句，不会重复创建。

@Cleanup

自动资源管理，无需关心close()方法的调用。

可以使用@Cleanup来让资源在代码作用域结束前被回收。可以像这样在局部变量申明前添加@Cleanup注解：@Cleanup InputStream in = new FileInputStream("some/file");。

如果想要清理的资源没有close()方法，但是有其它无参数清理方法，可以指定清理方法的名字：@Cleanup("dispose") org.eclipse.swt.widgets.CoolBar bar = new CoolBar(parent, 0);。

默认情况下，清理方法是close()。需要参数的清理方法不能使用@Cleanup。

Lombok

翻译成Java

@Getter and @Setter

不用再写Getter和Setter方法了。

可以在任何字段上添加@Getter或@Setter注解，Lombok会自动生成默认的getter/setter方法。

默认getter方法单纯返回字段的值，如果字段名是foo，getter方法名是getFoo（如果字段类型是boolean，getter方法名是isFoo）。如果字段名是foo，默认setter方法名是setFoo，返回类型是void，有1个和字段相同类型的参数，单纯把参数的值赋值给字段。

生成的getter/setter方法是public的，除非明确指定了AccessLevel。合法的访问等级有PUBLIC、PROTECTED、PACKAGE和PRIVATE。

也可以在类上添加@Getter或@Setter注解。这和在类中所有非静态字段上添加注解是一样的效果。

可以在任何字段上使用AccessLevel.NONE取消getter/setter的生成。这会覆盖类上的@Getter、@Setter或@Data注解。

@ToString

任何类的定义都可以使用@ToString注解，让lombok生成toString()方法的实现。默认情况下，它将打印类名，以及每个字段的值，用逗号分隔。

通过将includeFieldNames参数设置为true，可以让toString()方法在输出时添加字段名。

默认情况下，将打印所有非静态字段。如果想跳过一些字段，可以在exclude参数中指定，每个指定的字段都不会被打印出来。或者，可以在of参数中指定希望打印的字段。

通过将callSuper设置为true，可以将超类实现的toString输出内容包含到本类的输出。请注意，在java.lang.Object中默认的toString()实现几乎没有意义，因此不需要使用这个功能，除非正在扩展另一个类。

@EqualsAndHashCode

从对象的字段生成hashCode和equals的实现。

任何类的定义都可以用@EqualsAndHashCode注解，让lombok生成equals(Object other)和hashCode()方法的实现。默认情况下，它将使用所有非静态的、非transient的字段，但是也可以在注解的可选exclude参数中排除更多字段。或者，可以在of参数中指定想要使用的字段。

如果在继承了其他类的子类上使用@EqualsAndHashCode，这个特性会变得更加复杂。通常，为这种类自动生成一个equals和hashCode方法是一个坏主意，因为父类也定义了字段，它们也需要equals/hashCode代码，但这段代码不会生成。通过将callSuper设置为true，可以在生成的方法中包含父类的equals和hashCode方法。
